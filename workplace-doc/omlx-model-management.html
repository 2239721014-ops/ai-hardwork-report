01什么是 oMLX 模型管理？

oMLX 是一个专为 Apple Silicon Mac 优化的本地 LLM 推理服务器，其模型管理功能让你可以轻松管理多个 LLM、VLM、Embedding 和 Reranker 模型。

通过内置的 Web 管理面板，你可以：

  • 一键下载 HuggingFace 模型
  • 实时监控模型运行状态
  • 手动加载/卸载模型
  • 设置模型固定和 TTL
  • 配置采样参数和聊天模板

02核心功能

模型下载器
内置

直接从 HuggingFace 下载 MLX 格式模型，支持搜索、预览文件大小、一键下载。

分层 KV 缓存
热缓存

热缓存(RAM) + 冷缓存(SSD) 自动管理，模型上下文持久化，重启不丢失。

多模型服务
LRU

同时运行多个模型，LRU 自动驱逐，内存不足时自动卸载最久未使用的模型。

模型固定
VIP

常用模型可设置为"固定"，始终保持在内存中，不会被 LRU 驱逐。

03操作指南

1
打开管理面板

启动 oMLX 服务后，在浏览器访问 http://localhost:8000/admin

管理面板界面
2
进入模型管理页面

点击左侧导航栏的「Models」或「模型」选项

3
查看已加载模型

页面显示所有已加载的模型，包括：

  • 模型名称和类型 (LLM/VLM/Embedding)
  • 当前内存占用
  • 运行状态 (已加载/已卸载)
  • KV 缓存命中率
4
加载/卸载模型

点击模型卡片上的按钮进行操作：

  • 加载：点击加载按钮将模型加载到内存
  • 卸载：点击卸载按钮释放模型内存
  • 固定：点击图钉图标保持模型常驻内存
5
下载新模型

点击「Download Model」或「下载模型」按钮：

  1. 在搜索框输入模型名称 (如 Qwen3, Llama)
  2. 浏览搜索结果，查看模型大小
  3. 点击下载按钮等待完成
  4. 下载完成后自动出现在模型列表

04配置选项

常用 CLI 参数
# 模型目录
omlx serve --model-dir ~/models

# 内存限制
omlx serve --max-model-memory 32GB

# 启用 SSD 缓存
omlx serve --paged-ssd-cache-dir ~/.omlx/cache

# 批处理大小
omlx serve --prefill-batch-size 8 --completion-batch-size 32

# API 密钥
omlx serve --api-key your-secret-key
Per-Model 设置

在管理面板中，点击模型卡片可设置：

  • 采样参数 (Temperature, Top-P)
  • 聊天模板参数
  • TTL 空闲超时
  • 模型别名
  • 模型类型覆盖

集成 OpenClaw

在管理面板 → Integrations 一键配置 OpenClaw 连接：

API 地址: http://localhost:8000/v1
模型名称: 你加载的模型名

05常见问题

检查模型格式是否正确（必须是 MLX 格式）。确保 macOS 内存充足，可尝试减少 --max-process-memory 值。

在管理面板的模型卡片上显示当前内存占用。也可使用菜单栏应用查看实时内存使用情况。

SSD 缓存作为冷存储，当 RAM 缓存满时自动将 KV 块保存到 SSD。下次请求时从磁盘恢复，无需重新计算上下文。

  • LLM: mlx-lm 支持的所有模型
  • VLM: Qwen3.5, GLM-4V, Pixtral 等
  • OCR: DeepSeek-OCR, DOTS-OCR
  • Embedding: BGE-M3, ModernBERT
  • Reranker: ModernBERT, XLM-RoBERTa
